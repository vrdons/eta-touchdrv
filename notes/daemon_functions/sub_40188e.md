Programın **ana giriş noktası** (`main` fonksiyonu). Komut satırı argümanlarını ayrıştırır, cihazı açar ve ana döngüyü çalıştırır.

### Argüman ayrıştırma (`getopt_long` döngüsü)

- Seçenek string'i: `"e:d:vVh"`.
- Uzun seçenekler [[data_4ef5c0]] tablosundan okunuyor.
- **`'e'` (0x65):** [[data_4fa320]] (`optarg`) değeri `var_18`'e atanıyor (ellipse backend adı).
- **`'d'` (0x64):** [[data_4fa320]] (`optarg`) değeri `var_20`'ye atanıyor (debug log yolu).
- **`'v'` (0x76):** [[data_4f5310]] global sayacı 1 artırılıyor (verbosity seviyesi).
- **`'V'` (0x56):** [[sub_434970]] ile versiyon yazdırılıyor (`"%d.%d.%d.%d\n"`), `return 0`.
- **`'h'` (0x68):** [[sub_4017a1]] çağrılıyor (yardım mesajı), `return 0`.
- **Bilinmeyen seçenek:** [[data_4f3708]] (stderr) üzerine `"Try '%s --help' for usage.\n"` yazdırılıp `return 1`.

### Başlatma aşaması (argüman ayrıştırma sonrası)

1. [[sub_434970]] ile `"OpticalTouchDevice: version %d.%d.%d.%d\n"` yazdırılıyor.
2. [[sub_433bc0]] çağrılıyor, argüman olarak [[sub_401770]] veriliyor (muhtemelen sinyal handler kaydı).
3. [[sub_432c60]] iki kere çağrılıyor:
    - İlk: sinyal `0xF` (SIGTERM) için [[sub_401781]] handler olarak kaydediliyor.
    - İkinci: sinyal `0x2` (SIGINT) için [[sub_401781]] handler olarak kaydediliyor.
4. [[sub_41ed21]] çağrılıyor, `var_18` (ellipse backend) argüman olarak veriliyor.
5. `var_20` null ise `getenv("TOUCH_DEBUG_LOG")` ile ortam değişkeninden okunuyor.
6. [[sub_4201b4]] çağrılıyor, `var_20` (debug log yolu) argüman olarak veriliyor.
7. [[sub_447650]] ile `0xBBB0` byte'lık bellek ayrılıyor (ana context yapısı). Başarısız olursa `"FATAL: malloc failed"` yazılıp `return 1`.
8. [[sub_4010b0]] (memset) ile `var_78` (0x40), `var_178` (0x100) ve context yapısı (0xBBB0) sıfırlanıyor.

### Ana döngü

- [[data_4f5314]] sıfır olduğu sürece döngü devam ediyor (sinyal handler'ın set ettiği shutdown flag'i).
- `usleep(0)` ile yield yapılıyor.

#### Cihaz açık değilse (`var_9_1 == 0`):

- `0` ile `0x1E` (30) arası bir `for` döngüsünde cihaz yolu deneniyor:
    - [[sub_434a40]] (snprintf) ile `"/dev/OtdUsbRaw%03d"` formatında yol üretiliyor.
    - [[sub_464110]] (open) ile `O_RDWR` (2) modunda açılmaya çalışılıyor.
    - File descriptor `*rax_21`'e yazılıyor.
    - Başarılı olursa (`>= 0`) döngü kırılıyor.
- Cihaz açılamadıysa: `usleep(0xF4240)` (1 saniye) bekleniyor.
- Cihaz açıldıysa:
    - `"open device \"%s\".\n"` yazdırılıyor.
    - [[sub_403299]] çağrılıyor (cihaz başlatma/init). Argümanlar: context yapısının `0x2F2 * 4 = 0xBC8` offset'indeki alan ve context pointer'ı.
    - Init başarılıysa (`!= 0`):
        - `var_9_1 = 1` (cihaz hazır flag'i).
        - `"ready."` yazdırılıyor.
    - Init başarısızsa:
        - [[sub_4643b0]] (close) ile cihaz kapatılıyor.
        - `"initializing failed and close device."` yazdırılıyor.
        - `usleep(0xF4240)` bekleniyor.

#### Cihaz açıksa (`var_9_1 == 1`):

- [[sub_464240]] (read) ile `*rax_21` file descriptor'ından `var_78`'e `0x40` byte okunuyor.
- Tam `0x40` byte okununca:
    - [[sub_403370]] çağrılıyor: okunan paketi context yapısının `0xBC8` offset'indeki alana işliyor.
- Okuma hatası (`< 0`) olursa:
    - [[sub_4643b0]] (close) ile cihaz kapatılıyor.
    - `var_9_1 = 0` yapılıyor (tekrar arama moduna dönüş).
    - `"close device."` yazdırılıyor.
    - `usleep(0xF4240)` bekleniyor.

### Temizleme (döngü çıkışı)

- `"Shutdown requested, cleaning up..."` yazdırılıyor.
- Cihaz açıksa (`var_9_1 != 0` ve `*rax_21 >= 0`): [[sub_4643b0]] (close) ile kapatılıyor.
- [[sub_447c00]] (free) ile context yapısı serbest bırakılıyor.
- `return 0`.
