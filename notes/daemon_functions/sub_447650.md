
- Bellek ayırma yardımcı fonksiyonu.
- Girdi `arg1` istenen boyuttur.
- Önce [[data_4fa2e0]] kontrol edilir.
- [[data_4fa2e0]] == 0 ise başlatma yolu çalışır:
- [[sub_443fb0]] çağrılır.
- Sonra `arg1 < 0` ise hata verilir, `fsbase - 0x48` alanına `0xc` yazılır ve `0` döner.
- `arg1 >= 0` ise ana ayırma akışına girer.

- Küçük boyutlu istekler için thread-local cache yolu denenir.
- Boyuttan sınıf/index türetilir: `rbx_1`.
- `fsbase - 0x40` üzerinden tcache benzeri thread-local yapı alınır.
- Thread-local yapı yoksa ve `fsbase - 0x38 == 0` ise [[sub_446f00]] ile thread yapısı hazırlanır.
- Eğer index [[data_4f37c8]] sınırından küçük ve ilgili cache sayacı sıfır değilse:
- İlgili cached chunk alınır.
- Chunk işaretçisi alt bit kontrolü başarısızsa [[sub_443510]] çağrılır.
- Başarılı durumda encoded next pointer çözülür/güncellenir.
- İlgili sayaç bir azaltılır.
- `*(rax_5 + 8) = 0` yapılır.
- Chunk pointer döndürülür.

- Tcache yolu başarısızsa arena yolu çalışır.
- [[data_4f40a4]] != 0 ise doğrudan [[data_4f3800]] üstünden [[sub_446070]] çağrılır.
- Dönüş `0` ise ayırma başarısızdır, `0` döner.
- Dönüş geçerliyse chunk metadata kontrol edilir.
- Eğer arena/heap tutarlılığı uygunsa pointer döndürülür.
- Tutarsızlık varsa [[sub_43d4f0]] ile `"Fatal glibc error: malloc assertion failure in %s: %s\n"` yazdırılır.

- [[data_4f40a4]] == 0 ise thread arena yolu kullanılır.
- `fsbase - 0x30` içinden arena lock/yapı pointer alınır.
- Arena pointer yoksa:
- [[sub_444430]] ile arena seçilir/oluşturulur.
- Sonra [[sub_446070]] ile ayırma denenir.
- Başarısızsa ve arena varsa `arena_get_retry` çağrılır.
- Yeni arena ile [[sub_446070]] tekrar denenir.

- Arena pointer başta varsa lock alınır.
- `*rbx_1 == 0` ise `1` yapılarak lock alınır.
- Zaten doluysa `[[sub_4427d0]]` ile bekleme/yavaş yol çalışır.
- Lock alındıktan sonra `[[sub_446070]]` ile ayırma yapılır.
- Sonra lock bırakılır: `*rbx_1 = 0`.
- Önceki lock değeri `> 1` ise `__lll_lock_wake_private(rbx_1)` çağrılır.

- Ayırma sonrası chunk metadata doğrulanır.
- `(*(rax_4 - 8)).d & 2` set ise pointer doğrudan geçerli sayılır.
- Aksi halde `& 4` biti ile arena tabanı/owner bilgisi kontrol edilir.
- Sahip arena `rbx_1` ile eşleşirse pointer döner.
- Eşleşmezse `[[sub_43d4f0]]` ile fatal malloc assertion hatası verilir.

- Hata yolu:
- Negatif boyutta `*(fsbase - 0x48) = 0xc` yapılır.
- `0` döner.
